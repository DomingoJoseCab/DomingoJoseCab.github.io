4.1 Variables Difusas¶

Para simplificar la implementacin de las variables difusas se utilizarn conjuntos difusos triangulares. A continuacin, veremos qu atributos y qu funciones conforman esta clase.

Los atributos de la clase sern los siguientes:

  • universe: El universo del discurso de la variable. Este universo ser el array ordenado de todos los posibles valores que puede tomar la variable dada.
  • memb_func: Un diccionario con los grados de pertenencia de cada elemento del universo del discurso para cada uno de los conjuntos difusos que se generen de la variable difusa. Dicho diccionario lo obtendremos de la funcin _membership_functions.
  • labels: Etiquetas de cada uno de los conjuntos difusos, que estarn repartidas entre: Low, Mid y High.
  • name: Nombre de la variable difusa.

Las funciones de la clase sern las siguientes:

  • _membership_functions: Funcin interna para generar el diccionario con las distintas funciones de pertenencia.
  • fuzzify: Funcin que devuelve el mejor conjunto difuso para un valor dentro del universo del discurso.
  • plot: Funcin que muestra de manera grfica los conjuntos difusos de la variable.

Posteriormente, explicaremos el cdigo implementado en cada apartado para lograr un correcto funcionamiento. Pero antes de nada hay que tener en cuenta que para utilizar la clase, habr que calcular previamente los tringulos de los conjuntos difusos. Para ello, se utilizar la estructura fuzzy_points, que es diccionario con la definicin de los tringulos y forma {label: [start, peak, end]} para cada tringulo.

Una vez entendido el concepto y la estructura que conforman las variables difusas podemos comenzar a explicar la implementacin del cdigo. Dicha explicacin la realizaremos conforme aparece el cdigo en nuestra clase con comentarios.

import numpy as np
import pandas as pd
import skfuzzy as fuzz
import matplotlib.pyplot as plt

class FuzzyVariable:
    def __init__(self, name, variable, fuzzy_points):
        
        # Para comenzar explicaremos como obtener el universo de nuestra variable: para obtener dicho universo
        # debemos ordenar aquellos valores nicos que puede tomar la variable, para ello hacemos uso de numpy.
        self.universe = np.array(np.sort(np.unique(variable)),dtype='float64')
        
        # Para obtener el diccionario con los grados de pertenencia realizaremos una llamada a la funcin _membship_functions.
        self.memb_func = self._membership_functions(self.universe, fuzzy_points)
        
        # Las etiquetas las obtenemos del diccionario que define los tringulos.
        self.labels = fuzzy_points.keys()
        self.name = name
    
    def _membership_functions(self, variable, fuzzy_points):
        # Este diccionario ser en el que almacenemos los grados de pertenencia finales.
        fuzz_dict = {}
        
        # En este diccionario tendremos los correspondientes tringulos, para cada etiqueta tenemos
        # un tringulo con sus correspondientes puntos.
        pert = {}
        pert['Low'] = fuzz.trimf(self.universe,fuzzy_points['Low'])
        pert['Mid'] = fuzz.trimf(self.universe,fuzzy_points['Mid'])
        pert['High'] = fuzz.trimf(self.universe,fuzzy_points['High'])
        
        # A continuacin, obtendremos el grado de pertenencia de cada valor en la variable con cada
        # uno de los posibles tringulos. Para ello tendremos un diccionario para cada etiqueta donde
        # guardaremos el grado de pertenencia de cada valor de la variable.
        dict_pert_low = {}
        dict_pert_mid = {}
        dict_pert_high = {}

        for x in variable:
            # Por lo tanto, para cada valor de la variable obtenemos el grado de pertenencia con los tres
            # posibles tringulos, es decir, para cada etiqueta.
            pert_low = fuzz.interp_membership(self.universe,pert['Low'],x)
            pert_mid = fuzz.interp_membership(self.universe,pert['Mid'],x)
            pert_high = fuzz.interp_membership(self.universe,pert['High'],x)
            
            dict_pert_low[x] = pert_low
            dict_pert_mid[x] = pert_mid
            dict_pert_high[x] = pert_high
        
        # El resultado final de recorrer todos los posibles valores ser los grado de pertenencia en los 
        # posibles tringulos.
        fuzz_dict['Low'] = dict_pert_low
        fuzz_dict['Mid'] = dict_pert_mid
        fuzz_dict['High'] = dict_pert_high
        
        # Devolvemos el diccionario con los grados de pertenencia.
        return fuzz_dict
    
    def fuzzify(self, value):
        best = ''
        max_val = -1
        # Recorremos todas las posibles etiquetas para obtener la mejor.
        for label in self.labels: 
            # Obtenemos el grado de pertenencia del valor para cada posible etiqueta y la comparamos con la 
            # mejor hasta el momento. Por lo tanto, al final de recorrer todas las etiquetes obtendremos 
            # aquella etiqueta donde el valor pasado tiene el mayor grado de pertenencia.
            aux_val = self.memb_func[label][value]
            if aux_val > max_val:
                max_val = aux_val
                best = label
        return best

    def plot(self):
        fig, ax0 = plt.subplots(nrows=1)
        for label in self.labels:
            ax0.plot(self.universe, self.fuzz_dict[label], linewidth=1.5, label=label)
        ax0.set_title(self.name)
        ax0.legend()
        plt.tight_layout()

4.2 Transformador Fuzzy¶

Deberemos implementar un transformador para llevar a cabo la fuzzificacin de la base de datos, dicho transformador se encargar de generar las variables difusas a partir de las variables continuas. La implementacin consiste en generar las variables difusas necesarias y utilizar la funcin fuzzify para fuzzificar los valores.

Para ello comenzaremos explicando cmo funciona el transformador, cules son sus atributos y cules son sus funciones. Los atributos de la clase son los siguientes:

  • sets: Nmero de conjuntos difusos para cada variable, por defecto son 3.
  • labels: Etiquetas difusas para los conjuntos, por defecto son Low, Mid y High.

La clase se dividir en las funciones:

  • fit: encargado de aprender las variables difusas a partir de los datos.
  • transform: crear una nueva base de datos sustituyendo los valores continuos por las etiquetas de las variables.
  • fit_transform: auxiliar para hacer los dos procedimientos en un solo paso.
  • _equal_width: Esta funcin coge una variable en formato NumPy y genera cortes de igual anchura. Es decir, si la variable es [1,2,3,4,5,6,7] y tiene que hacer 3 cortes, devuelve los puntos [1,4,7].
  • _expand_points: Esta funcin coge los puntos de equal_width y los expande para generar los tringulos en el formato [start, peak, end].

A continuacin, explicaremos el cdigo implementado conforme aparece en la prctica.

from sklearn.base import BaseEstimator

# Ejemplo de la estructura bsica de un algoritmo que hereda de BaseEstimator siguiendo la estructura de SciKit
class FuzzyEncoder(BaseEstimator):
    # constructor
    def __init__(self, sets=3, labels=['Low', 'Mid', 'High']):
        self.fuzzy_vars = {}
        self.sets = sets
        self.labels = labels

    # siguiendo las guias de scikit-learn, creamos las funciones fit para el entrenamiento
    def fit(self, X):
        
        # Para aprender las variables difusas a partir de los datos tenemos que realizar el entrenamiento para 
        # as poder realizar una transformacin con lo aprendido. Dicho entrenamiento consistir en generar todas
        # aquellas variables difusas posibles a partir de las variables continuas de una base de datos de entrenamiento.

        for n in range(len(X[0,:])):
            # Comprobamos que la variable seleccionada sea continua y calculamos los fuzzy points de dicha variable
            # para posteriormente crear la variable difusa. Realizando as el entrenamiento con cada variable continua.
            if type(X[0,n]) == float :
                fuzzy_points = self._expand_points(self._equal_width(X[:,n],self.sets),self.labels)
                self.fuzzy_vars[features[n]] = FuzzyVariable(features[n],X[:,n],fuzzy_points)
        return self
        
    # y transform para aplicar lo aprendido y transformar la base de datos creando una nueva
    def transform(self, X):
        new_X = X.copy()
        
        # A continuacin, tenemos que aplicar lo aprendido anteriormente por lo tanto tenemos que recorrer otra vez la base 
        # de datos obteniendo la fuzzificacin para cada valor de cada variable.
        for n in range(len(new_X[0,:])):
            
            # Comprobamos que la variable haya sido entrenada previamente y realizamos la transformacin. Para ello almacenamos
            # los ndices de la variable y calculamos la fuzzificacin para cada valor con fuzzify. Por ltimo, utilizando la 
            # funcin put que nos proporciona numpy modificamos los valores continuos por los difusos.
            
            if features[n] in self.fuzzy_vars:
                indices = np.arange(0,len(new_X[:,n]),1)
                fuzzificadas = [self.fuzzy_vars[features[n]].fuzzify(x) for x in new_X[:,n]]
                new_X[:,n].put(indices,fuzzificadas)
            
        return new_X
    
    def fit_transform(self, X):
        return self.fit(X).transform(X)

    def _equal_width(self, var, sets):
        try:
            cut = pd.cut(var, sets - 1)
            sol = [cat.left for cat in cut.categories] + [cut.categories[-1].right]
        except ValueError: 
            cut = pd.cut(var, sets - 1, duplicates='drop')
            sol = [var.min()] + [cat.left for cat in cut.categories] + [cut.categories[-1].right]
        sol[0] = var.min()
        return sol

    def _expand_points(self, fp, labels):
        ep = {}
        expanded_points = [fp[0]] + fp + [fp[-1]]

        for i in range(len(expanded_points) - 2):
            ep[labels[i]] = [expanded_points[i], expanded_points[i+1], expanded_points[i+2]]

        return ep

5. Motor de inferencia¶

Finalmente creamos el sistema basado en reglas difusas. Vamos a tener una clase abstracta general y una clase que extienda de esta y genera las reglas con el algoritmo Wang y Mendel.

La clase abstracta RuleBasedSystem se dividir en las funciones:

  • fit: Metodo abstracto que se programar en la clase que extienda de RuleBasedSystem.
  • _create_rbs: encargado de aprender el sistema basado en reglas. Tiene los siguientes parmetros:
    • rule_list: Lista de reglas con las que se crear el sistema de control
    • fuzzy_variables: Variables difusas extraidas de FuzzyEncoder.fuzzy_vars.
    • discrete_variables: Diccionario con las variables discretas.
    • target_variable: Diccionario con la variable objetivo.
  • predict: Devuelve la clasificacin para una instancia de la base de datos segun el sistema basado en reglas.
  • score: Devuelve el score de la base de datos.

                # Y concatenamos todos los antecedentes de la regla.
                else:
                    antecedents = antecedents & antecedents_var[rule[0][ante][0]][rule[0][ante][1]]
            
            # Finalmente aadimos el consecuente almacenado en rule[1] y aadimos la regla.
            rule_ = ctrl.Rule(antecedent=(antecedents), consequent=consequent[rule[1]])
            rules.append(rule_)
        
        # Por ltimo, aadimos todas las reglas al sistema de control.
        self.ctrl = ctrl.ControlSystem(rules=rules)
        
    @abstractmethod
    def fit():
        pass
    
    def predict(self, x):
        sim = ctrl.ControlSystemSimulation(self.ctrl, flush_after_run=1000)
        # Para predecir, hay que aadir los valores de la instancia al sistema
        # de simulacion de control. Primero hay que ver si existe el antecedente 
        # en el sistema de control (posiblemente las reglas extraidas del arbol
        # no tengan todas las caracteristicas de la instancia). En caso afirmativo,
        # la variable 'temperatura' = 8.7 se aade como sim.input['temperatura'] = 8.7    
        
        # ----------------------------------------------------------------------------------
        
        # Recorremos todas las variables comprobando si existe el antecedente o no, en caso afirmativo,
        # aadimos el valor correspondiente x[n].
        for n in range(len(x)):
            try:
                sim.input[attributes[n]] = x[n]
            except:
                continue
                
        # Despues se calculan los grados de las reglas
        sim.compute()

        # Finalmente se devuelve el resultado que se encuentra en sim.output['target']
        # si la variable a predecir se llama 'target'.
        return sim.output[target]

    def score(self, X, y):
        all_predictions = np.array([[round(self.predict(x))] for x in X])
        return np.sum(all_predictions==y)/y.shape[0]

6. Extracción de reglas Wang y Mendel¶

La clase WangMendelRuleBasedSystem se encargar de obtener las reglas mediante el algoritmo de Wang y Mendel. En esta clase podemos encontrar las siguientes funciones:

  • _extract_rules: encargado de procesar la base de datos y obtener las reglas. Para esta funcin hemos aadido un parmetro, se trata de selected_variables, para el filtrado de variables dado que si no realizamos un filtrado obtendriamos una gran cantidad de reglas que el sistema no soporta.
  • fit: Mtodo que entrena el sistema basado en reglas.
In [13]:
class WangMendelRuleBasedSystem(RuleBasedSystem):
    def __init__(self):
        super().__init__()

    # Se recomienda este formato para extraer las reglas
    # del arbol, ya que simplifica la manera de tratar los 
    # antecedentes y consecuentes a la hora de generar el 
    # sistema basado en reglas
    
    def _extract_rules(self, X, y, fuzzy_variables, selected_variables):
        
        # Tenemos un diccionario donde almacenamos el grado de recubrimiento para cada regla.
        dict_rules = {}
        
        # Recorremos todas las filas de la base de datos.
        for n in range(len(X)):
            degree = 1
            antecedents = []
            
            # Para cada fila creamos la regla correspondiente.
            for j in range(len(attributes)):
                
                # Comprobamos que la variable esta entre las seleccionadas para 
                # realizar el filtrado.
                if attributes[j] in selected_variables:
                    
                    # Y comprobamos si se trata de una variable difusa o discreta. En el
                    # caso de ser difusa tendremos que actualizar el grado de pertenencia.
                    # Para discretas no hara falta ya que siempre sera 1.
                    if attributes[j] in fuzzy_variables:
                        fuzzy_val = fuzzy_vars.fuzzy_vars[attributes[j]].fuzzify(X[n,j])
                        degree *= fuzzy_vars.fuzzy_vars[attributes[j]].memb_func[fuzzy_val][X[n,j]]

                        ante = (attributes[j], fuzzy_val)

                    else:
                        ante = (attributes[j], X[n,j])     
                    
                    # Guardamos el antecedente junto con el resto.
                    antecedents.append(ante)
            
            # Creamos la regla de la forma: ((antecedent),consequent) para poder manejar
            # mejor que reglas estn o no en el diccionario.
            
            rule = (tuple(antecedents), y[n][0])
            
            # Comprobamos si los antecedentes de la regla estn en el diccionario. Si no 
            # estn se aade directamente la regla al diccionario. En caso contrario de que
            # s esten, tendremos que calcular cul es la regla con mayor grado de recubrimiento,
            # ya que esa debe permanecer.
            
            if ((rule[0], 0) in dict_rules) or ((rule[0], 1) in dict_rules):
                try:
                    if degree > dict_rules[(rule[0], 1)]:
                        del(dict_rules[(rule[0], 1)])
                        dict_rules[rule] = degree
                except:
                    try:
                        if degree > dict_rules[(rule[0], 0)]:
                            del(dict_rules[(rule[0], 0)])
                            dict_rules[rule] = degree
                    except:
                        continue

            else:
                dict_rules[rule] = degree

        
        # Obtenemos todas las reglas del diccionario con .keys()
        dict_keys = dict_rules.keys()
        
        # Y modificamos el formato de las reglas: ([antecedents],consequent) para que 
        # nuestro sistema basado en reglas funcione correctamente.
        all_rules = []
        for x in dict_keys:
            rule_ = (list(x[0]),x[1])
            all_rules.append(rule_)
        
        # Para el algoritmo de Wang y Mendel, que se debe realizar en esta funcion
        # Ss recomienda el uso de un diccionario del tipo {rule: membership}
        # para que a la hora de almacenar las reglas y comprobar si hay que actualizar
        # el grado de pertenencia, sea lo mas eficiente posible.
        # Recordad que para que sea compatible con el algoritmo de la practica anterior,
        # habra que transformar ese diccionario a una lista antes de devolverlo.

        # fuzzy_variables sera un subconjunto de las variables difusas del problema, 
        # ya que un problema con tantas instancias y variables puede resultar muy 
        # complejo de extraer todas las reglas

        return all_rules


    def fit(self, X, y, fuzzy_variables, discrete_variables, target_variable, selected_variables):
        rule_list = self._extract_rules(X, y, fuzzy_variables, selected_variables)
        self._create_rbs(rule_list, fuzzy_variables, discrete_variables, target_variable)
